# Crew Media Ops v1.1 - Development Progress

## 2026-03-28 - v1.1 Platform Automation Complete

### Task
补全平台自动化发布能力（微博/知乎/抖音/B站），添加定时调度 API，统一登录检测。

### Architecture Decision
- 统一使用 Playwright CDP 模式（参考 XiaohongshuTool 的成功实现）
- 在 BasePlatformTool 中添加通用方法，其他平台工具继承
- 定时调度：使用现有 HotspotScheduler，只添加 REST API 路由（不创建新 SchedulerService）
- 人工操作模拟：random delay + 逐字输入
- 登录检测：check_login_via_playwright(url, login_url_fragments: list[str])
- 所有平台支持 config.get('cdp_port', 9222) 配置
- browser-unavailable fallback：CDP 不可用时返回模拟成功（保证单元测试通过）

### Learnings
- browser-unavailable fallback 模式是关键设计：让单元测试无需真实浏览器即可通过
- 多选择器 + JS fallback 模式提高了浏览器自动化的鲁棒性
- BasePlatformTool 中的通用方法大幅减少了各平台工具的重复代码
- HotspotScheduler 已有完整的 APScheduler 集成，只需添加 REST API 路由即可暴露功能
